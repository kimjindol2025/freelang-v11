## 2. 보고 규칙 (강제)

### 2.1 주장에는 증거
- "X 작동한다" / "X 완료" / "X% 달성" 같은 문장은
  **실행 로그 / HTTP 응답 / sha256 / diff 결과** 중 하나 이상을 커밋·문서 내
  인용해야 함.
- 증거 없으면 단정하지 말고 "관측되지 않음" / "재현 필요" 로 표기.

### 2.2 수치는 실측 산출물만
- 테스트 스크립트가 출력한 숫자만 인용.
- "감", "추정", "약" 수치 금지.
- 추정이 꼭 필요하면 `est=X basis=이유` 형식으로 추정임을 명시.

### 2.3 "완료" 단어 조건
다음이 모두 충족될 때만 사용 가능:
- 테스트 스크립트 작성됨
- 스크립트 실행 → PASS 관측
- 관측된 출력이 `parity-report.json` 또는 `tests/evidence/` 에 저장됨
- 커밋 메시지에 `evidence=<path>` 또는 동등 필드 포함

### 2.4 환경 상태는 실측 후 서술
- `ls <구체파일>`, `git status`, `git diff --diff-filter=D --name-only HEAD`
  같은 **직접 관측 명령** 먼저 실행.
- "파일이 없다 / 있다" 는 그 명령 출력 인용으로만.

### 2.5 시간·규모 추정 금지
- "수개월", "1~2주", "하루" 같은 시간 표현 사용 금지.
- AI 작업 단위는 세션의 토큰/커밋 수이지 벽시계 시간이 아님.
- 필요하면 "이번 세션에서 커밋 N개 예상" 처럼 산출물 단위로만.

### 2.6 축소 제안 금지
- 사용자 지시가 전체 범위면 **그 범위대로 착수**.
- 본인 판단으로 "MVP만", "일부만" 제안 금지.
- 범위 조정 요청은 사용자만 할 수 있음.

### 2.7 결정 회피 금지
- 본인이 관측과 판단으로 내릴 수 있는 결정은 직접 내림.
- 질문으로 되돌리는 경우는 **사용자만이 정할 수 있는 값** (디자인 방향,
  우선순위 등) 일 때로 한정.

### 2.8 과거 거짓 발견 시 공개 정정
- 기존 커밋·문서·메모리에서 본인이 만든 과장/거짓을 발견하면
  **즉시 정정 기록** 남김 (`CORRECTIONS.md` 또는 메모리 갱신).
- "이전 주장 X는 Y 증거에 의해 철회됨" 명시.

## 3. 커밋 메시지 필수 필드 (카테고리별)

### 구현 커밋
```
evidence=<test_script_path>
observed=<JSON 한 줄 또는 key=value>
test_cmd=<재현 명령>
```

### 검증/측정 커밋
```
measured=<metric>:<value>
method=<measurement_command>
```

### 정정 커밋
```
corrects=<prior_commit_sha>
reason=<falsified_claim>
```

## 4. 금지 표현 목록 (정규식 감지 대상)

- 단정: `완료됐다`, `작동한다`, `준하는`, `호환`, `파리티 달성`
- 수치 과장: `~급`, `수준`, `정도`, `쯤`
- 규모 겁주기: `수개월`, `수주`, `한 세션 내`, `며칠`
- 회피: `옵션 1/2/3`, `어떻게 할까요`, `결정 주세요`
  (단, 진짜 사용자 결정이 필요할 때만 예외)

AI 출력 전에 이 표현 자가 검열. 발견 시 증거 기반 표현으로 대체.
