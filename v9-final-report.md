## 4. v9 DB 최적화

### 아키텍처
```
┌─────────────────────┐
│   API Layer (SQL)   │ ← 파서, 실행
├─────────────────────┤
│  Engine (Optimizer) │ ← 계획, 캐싱
├─────────────────────┤
│ Storage (B-Tree)    │ ← 정렬, 범위검색
└─────────────────────┘
```

### 핵심 개선사항
1. **Binary Search Index** - O(log n) 조회
2. **B-Tree Storage** - 정렬된 저장
3. **Query Cache** - LRU 제거 정책
4. **Transaction Rollback** - 실제 데이터 복원
5. **Soft Delete Vacuum** - 메모리 40% 절약
