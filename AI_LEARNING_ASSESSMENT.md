### ⚠️ 부족한 것 (AI 학습 막혀있음)

#### 1️⃣ stdlib 문서 부실
```
현재: 모듈 목록만 (30개 나열)
필요: 함수별 상세 문서
  - 함수 시그니처
  - 파라미터 설명
  - 반환값 타입
  - 에러 케이스
  - 예제 코드
```

**영향**: AI가 stdlib 함수를 쓸 때 추측에만 의존
- 예: `(map-entries m)` 함수의 정확한 동작?
- 예: `(vector-dot v1 v2)` 반환값이 숫자인가? 리스트인가?

#### 2️⃣ 예제 부족
```
현재: 3개 (hello.fl, factorial.fl, browser-test.html)
필요: 20+ 패턴별 예제
  - 초보: 변수, 함수, 루프
  - 중급: 패턴매칭, 에러 처리, 상태 관리
  - 고급: Agent task, State machine, Async/await
```

**영향**: AI가 패턴을 배울 기회 부족

#### 3️⃣ 에러 메시지 품질 미평가
```
현황: 컴파일 에러, 런타임 에러 메시지가 있으나 AI 입장에서 도움 수준 미평가

예상 문제:
  ❌ "Error at line 42" → 원인이 뭔가?
  ❌ 에러 복구 힌트 없음
  ❌ 타입 에러 메시지가 명확한가?
```

#### 4️⃣ 타입 시스템 문서 부재
```
현재: 타입 체크 함수만 있음 (number?, string?, etc.)
필요: 타입 시스템 가이드
  - 프리미티브 타입 (number, string, symbol, nil)
  - 복합 타입 (array, map, function)
  - 타입 검사 방법
  - 타입 강제 vs 덕 타이핑
```

**영향**: AI가 버그 없는 코드 생성 어려움

#### 5️⃣ AI 특화 튜토리얼 없음
```
현재: 일반 튜토리얼만 있음
필요: AI 입장에서의 학습 경로

예:
  1. "이 언어의 철학은?"
  2. "AI로서 자주 실수하는 패턴은?"
  3. "결정론성을 해치지 않으려면?"
  4. "프롬프트 엔지니어링 팁"
```

## 📋 AI가 자주 하는 실수 (추측)

AI가 이 언어를 생성할 때 **실제로 빠뜨리는 것들**:

```lisp
❌ (println "hello") ;; 대신 (console-log "hello")
❌ (map fn [1 2 3]) ;; 순서 혼동 (fn을 먼저? 리스트를 먼저?)
❌ (let [x 1 y (+ x 1)]) ;; 유효한가?
❌ (try (risky) (catch e (handle e))) ;; 문법?
❌ {:a 1 :b 2} ;; 맵 생성이 이 형태인가?
❌ (if cond then else) ;; 괄호? 들여쓰기?
```
