# 행동 강령 (Code of Conduct)

> FreeLang 커뮤니티의 모든 참여자를 위한 행동 강령

---

## 우리의 약속

FreeLang 커뮤니티는 모든 사람에게 괴롭힘 없는 경험을 제공하기 위해 약속합니다.

우리는 다음을 약속합니다:

- ✅ 모두에게 **개방적이고 포용적인** 환경 제공
- ✅ **나이, 신체 크기, 장애, 민족성, 성 정체성, 경험 수준**을 초월한 환영
- ✅ **전문성 있고 존경하는** 교류 촉진
- ✅ **건설적인 피드백**과 개선 제시

---

## 우리의 기준

### 권장되는 행동

- 📢 다양한 관점 환영하기
- 🤝 다른 커뮤니티 멤버와 협력하기
- 🎯 건설적인 비판 제시하기
- 😊 친절하고 정중한 의사소통
- 👂 경청하고 이해하려 노력하기
- 🌟 타인의 성공을 축하하기

### 용인되지 않는 행동

- ❌ 혐오적인 언어나 폭력
- ❌ 인신공격이나 모욕
- ❌ 성적 괴롭힘이나 판단
- ❌ 사생활 침해
- ❌ 스팸이나 광고
- ❌ 차별이나 혐오
- ❌ 협박이나 폭력 언어

---

## 적용 범위

이 행동 강령은 다음 상황에 적용됩니다:

- 📝 GitHub Issues / Pull Requests 
- 💬 토론 포럼
- 📧 이메일 및 메시지
- 🎤 커뮤니티 이벤트 (온라인/오프라인)
- 📱 소셜 미디어 (FreeLang 관련)

---

## 행동 강령 위반 보고

### 문제 발생 시

1. **직접 대면 시도** (안전한 경우)
   - 당사자에게 행동이 부적절함을 알리기
   - 대부분의 경우 오해일 수 있음

2. **개인 메시지**
   - 모더레이터에게 상황 설명
   - 증거/로그 첨부

3. **공식 신고**
   - conduct@freelang.dev로 이메일
   - 제목: [REPORT] 상황 설명
   - 본문: 상황, 참여자, 증거

### 신고 템플릿

```
제목: [REPORT] Issues #123에서의 부적절한 언어

날짜: 2026-04-19
위치: Issues #123, 댓글 #456
관련자: @username

상황 설명:
[구체적인 상황 설명]

인용:
"문제가 되는 메시지"

행동 강령 위반:
[어느 부분이 위반되었는지]

요청 사항:
[처리 요청]
```

## 신고 처리

### 조사 과정

1. **접수**: 신고 내용 검증
2. **조사**: 관련자 인터뷰 및 증거 수집
3. **판단**: 행동 강령 위반 여부 판단
4. **조치**: 필요한 조치 실행
5. **통보**: 신고자/관련자 통보

### 가능한 조치

| 수준 | 설명 | 처리 |
|------|------|------|
| 1️⃣ **경고** | 경미한 첫 위반 | 비공개 경고 |
| 2️⃣ **일시 제한** | 반복된 위반 | 1주 커뮤니티 제한 |
| 3️⃣ **영구 차단** | 심각한 위반 | 커뮤니티 퇴출 |

## 모더레이터 역할

### 책임

- 📋 커뮤니티 가이드라인 시행
- 🛡️ 안전하고 존경하는 환경 유지
- ⚖️ 공정한 중재
- 🤝 건설적인 논의 유도

### 모더레이터 윤리

- ✅ 공정성과 투명성
- ✅ 이해 충돌 회피
- ✅ 기밀성 유지
- ✅ 신속한 대응

## 특수 상황별 가이드

### 정치 논의

```
✅ 허용:
  - FreeLang 정책/설계 논의
  - 기술 결정 이유 설명
  - 건설적인 의견 제시

❌ 금지:
  - 정치인/정당 비판
  - 정치 성향 강요
  - 부정적 일반화
```

### 기술 논쟁

```
✅ 허용:
  - 기술적 장단점 논의
  - 다른 접근법 제안
  - 데이터 기반 논증

❌ 금지:
  - 개인 공격 ("당신은 멍청하다")
  - 모욕 ("당신의 아이디어는 쓰레기")
  - 조롱 ("ㅋㅋㅋ 뭐하는 거야")
```

### 마이너리티 보호

```
특히 주의:
  - 젠더, 인종, 종교에 관한 고정관념
  - 신경다양성에 대한 부정적 언어
  - 성 정체성에 대한 오해
  - 장애인에 대한 차별

예시:
  ❌ "여자는 코딩을 못해"
  ❌ "자폐인은 공감이 없다"
  ❌ "다문화는 문제다"
```

## 커뮤니티 리더의 책임

### Issues/PR 리뷰어

```
할 일:
  ✅ 건설적인 피드백 제공
  ✅ 코드 + 사람 모두 존중
  ✅ 배경 이해하기
  ✅ 감정적 표현 피하기

하지 말 것:
  ❌ 인신공격
  ❌ 일방적 거부 (설명 없이)
  ❌ 비꼬는 반응
  ❌ 무시하기
```

### 토론 참여자

```
할 일:
  ✅ 다양한 의견 환영
  ✅ 가정하지 않고 물어보기
  ✅ 자신의 실수 인정하기
  ✅ 배움에 집중하기

하지 말 것:
  ❌ 다른 의견 무시
  ❌ 가정 기반 비판
  ❌ 실수 변명하기
  ❌ 우월감 표현하기
```

## 예시와 설명

### 좋은 예시

```
Issue: "파싱 성능 개선 제안"

❌ 나쁜 댓글:
"이건 너무 느려. 무능한가?"

✅ 좋은 댓글:
"파싱 성능이 병목인 것 같습니다.
현재 O(n²)인데, O(n)으로 개선할 수 있을 것 같습니다.
[링크]에서 유사 사례를 찾았습니다.
피드백 부탁합니다."
```

### Pull Request 리뷰

```
❌ 나쁜 리뷰:
"이 코드는 형편없다. 다시 작성하세요."

✅ 좋은 리뷰:
"감사합니다. 몇 가지 개선 제안입니다:

1. 이 부분 [링크]은 O(n)으로 개선 가능합니다.
   제안: [코드]
   
2. 엣지 케이스 처리가 필요합니다:
   - 빈 배열
   - null 값
   
이 수정 후 다시 리뷰하겠습니다.
좋은 PR 감사합니다!"
```

## 자주 묻는 질문 (FAQ)

### Q: 누군가 나를 공격했어요
**A:** 
1. 깊게 숨을 쉬기
2. 대응하지 말기 (감정적 표현 회피)
3. conduct@freelang.dev로 신고하기

### Q: 우발적으로 실수했어요
**A:**
1. 빨리 사과하기
2. 이해하려 노력하기
3. 앞으로 주의하기

### Q: 비판이 과한 것 같아요
**A:**
1. 구체적인 문제 지적하기
2. 모더레이터에게 상담하기
3. 필요시 신고하기

### Q: "농담인데 왜 화내?"
**A:**
농담이어도:
- ➡️ 타인이 상처받으면 그건 농담이 아닙니다
- ➡️ "미안해. 다음부터 주의할게" 가 정답
- ➡️ 방어하지 말고 이해하기
